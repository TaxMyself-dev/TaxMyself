# סיכום — טבלת report-review לאחר סבב השינויים

תיעוד מצב נוכחי בלבד (לא תכנון) — נכתב אחרי סדרת שינויים על `frontend/src/app/pages/report-review/`:
המרת עריכה inline לפופאפ, הסרת עמודות מקור/פעולות/סטטוס, והתאמות עיצוב לסרגל
הפעולות ולרוחב הטבלה.

---

## 1. תצוגה כללית

- טבלה אחת (`app-generic-table`), עם toggle "תצוגה רגילה" / "תצוגה מקצועית"
  (נשמר לכל משתמש ב-localStorage; רואי חשבון/אדמינים מתחילים בברירת מחדל
  במצב מקצועי).
- הטבלה מצטמצמת לרוחב העמודות בפועל כשאין תצוגה מקדימה של מסמך פתוחה
  (`.dialog-body:not(.with-preview) { width: fit-content }`) — אין יותר
  שטח ריק מיותר בצד. כשנפתחת תצוגה מקדימה (אייקון העין), הטבלה חוזרת
  לפיצול הרוחב הקבוע (58% טבלה / 40% מסמך) כפי שהיה.

## 2. עמודות — תצוגה רגילה

1. תיבת סימון (בחירה לאישור מרוכז; מנוטרלת אם השורה לא ניתנת לאישור)
2. **ספק** — שם הספק + ריבוע "ספק חדש" (ירוק=יתווסף/אפור=לא) + אייקון
   שגיאה (⚠, בכישלון שמירה) + פרומפט כפילות רכה מתחתיו בעת הצורך
3. **סוג** — "תנועה" לשורת תנועת-בנק בלבד, או סוג המסמך (חשבונית/קבלה/
   חשבונית מס קבלה/חשבונית זיכוי/וכו') לשורת מסמך; "מסמך לא מזוהה" כברירת
   מחדל כשאין סוג מזוהה. (עמודה זו איחדה לתוכה את עמודת "מקור" הישנה.)
4. תאריך
5. סכום (+ סכום בש"ח בסוגריים לשורות במטבע זר)
6. קטגוריה
7. תת קטגוריה
8. % מע"מ
9. % מס
10. תקופה (תקופת דיווח; מודגשת בענבר אם נקבעה ידנית)

**לא מוצגות בתצוגה הרגילה**: מס' עוסק, מס' חשבונית, מס' הקצאה — קיימות
ונערכות רק דרך פופאפ העריכה, לא כעמודות בטבלה היומיומית.

## 3. עמודות — תצוגה מקצועית

1. תיבת סימון
2. ספק (כנ"ל)
3. סוג (כנ"ל)
4. מס' חשבונית
5. תאריך
6. **תיאור** — עמודת הסיווג האחידה של D7 (קטגוריה/תת-קטגוריה יחד),
   הורחבה מ-190px ל-260px כדי שלא תיצמד לעמודת הסכום
7. סכום
8. **כרטיס** — בורר כרטיס הנהלת חשבונות (בחירת כרטיס = הסיווג המלא, D1)
9. % מע"מ, % מס — קריאה בלבד, נגזרים מהכרטיס
10. % פחת — קריאה בלבד
11. תקופה

**לא קיימות עוד בכלל** (לא רק מוסתרות): עמודת "סטטוס" (הפס הצבעוני
מוכן/חסר מיפוי/וכו') וכפתורי ה"השלם מיפוי"/"למה ההוצאה שייכת?" שהיו
נלווים אליה — הוסרו לגמרי לפי בקשה מפורשת, לא רק הועברו למקום אחר.

## 4. סרגל פעולות (hover, מופיע כשעוברים עם העכבר מעל שורה)

מוצג כרצועה צפה (position: fixed) משמאל לשורה; כל כפתור: אייקון + טקסט
תמיד גלוי, **בלי** tooltip בהובר (הוסר כי היה כפול לטקסט הגלוי ממילא).
הרצועה מתאימה את גובהה לתוכן שלה (לא לגובה קבוע ישן של 76px), כדי שלא
תשתרע על יותר משורה אחת.

פעולות זמינות (כל אחת מותנית ב-`showWhen` לפי סוג/מצב השורה):

| פעולה | מתי מוצגת |
|---|---|
| ערוך הוצאה | כל שורה שאינה "מסמך שנתי" ואינה "לא מזוהה" — פותח את פופאפ העריכה |
| צפה במסמך לצד הטבלה | שורה עם מסמך מקושר (driveFileId), שאינה "מסמך שנתי" |
| מיין — קבע מה המסמך הזה | שורות D8 "לא מזוהה" — פותח פופאפ מיון |
| פצל | שורות מסוג invoice_receipt_pair |
| העלה מסמך חדש | שורות תנועה-בלבד |
| קשר למסמך קיים | שורות תנועה-בלבד, כשיש שורות "מסמך בלבד" זמינות — פותח פופאפ קישור |
| העבר לארכיון | כל שורה שאינה "מסמך שנתי" (לתנועה-בלבד מתנהג כמו מחיקה) |
| מחק | כל שורה שאינה "מסמך שנתי" |

## 5. פופאפ עריכת הוצאה (`app-report-review-edit-dialog`)

נפתח מפעולת "ערוך הוצאה". שני מצבי פריסה:
- **עם מסמך** (matched/doc_only) — שני טורים: טופס עריכה מימין, תצוגת
  Drive מקדימה משמאל, עם גלילה.
- **בלי מסמך** (תנועה-בלבד) — טור אחד ברוחב מלא, בלי אזור מסמך.

שדות משותפים: ספק, מס' עוסק, תאריך, סכום, קטגוריה/תת-קטגוריה (או כרטיס
בתצוגה מקצועית), % מע"מ, % מס, תקופת דיווח (+ "אחר" לתקופה מותאמת אישית).
שדות רק לשורות עם מסמך: סוג מסמך, מס' הקצאה, "הוסף ספק זה לרשימה".
צ'קבוקס "החל סיווג זה גם על שורות זהות בטבלה" (מסומן כברירת מחדל) —
שולט אם השינוי מופץ לשורות אחיות של אותו ספק בעת השמירה.

**שמירה** כותבת את השדות על השורה בזיכרון בלבד וסוגרת את הפופאפ — היא
**לא** קוראת לשום endpoint של אישור. האישור בפועל קורה רק דרך תיבת
הסימון + כפתור "אישור הוצאות נבחרות" בתחתית, או "שמור בכל זאת" על כפילות.
**ביטול** משליך את הטיוטה לגמרי — השורה עצמה לא נגעת.

## 6. פופאפים נוספים

- **תקופת דיווח מותאמת אישית** — נפתח מבחירת "אחר" בתקופה בתוך פופאפ
  העריכה.
- **התאמת נתוני ספק** — נפתח לפני אישור מרוכז אם יש כמה שורות לספק חדש
  זהה עם ערכים שונים.
- **קישור לתנועה** — נפתח מפעולת "קשר למסמך קיים"; בורר מסמך + אישור/ביטול.
- **מה המסמך הזה?** (מיון D8) — נפתח מפעולת "מיין"; שני כפתורים
  (חשבונית הוצאה / מסמך שנתי) + ביטול.

## 7. שורת תחתית (footer)

תיבת "בחר הכל" (רק שורות ניתנות לאישור), טקסט "נבחרו X מתוך Y ניתנות
לאישור", כפתורי "אישור הוצאות נבחרות" ו"דלג".

## 8. מה הוסר לגמרי בסבב הזה (לא רק שונה מיקום)

- עמודת "מקור" (אייקון סוג שורה) — מוזגה לתוך עמודת "סוג".
- עמודת "פעולות" הישנה כעמודת טבלה נפרדת.
- עמודת "סטטוס" + כל לוגיקת ה-badge שלה (D9 mapping status).
- יכולת "השלם מיפוי" (רו"ח) ו"למה ההוצאה שייכת?" (לקוח) — הוסרו לגמרי,
  לא הועברו למקום אחר.
- הסימון "נערכה" (עיפרון ליד שם הספק).
- עריכה inline בתוך התא (הוחלפה כליל בפופאפ).

## 9. מה הועבר למקום אחר (לא הוסר)

- אייקון כישלון (⚠) ופרומפט כפילות רכה — עברו מעמודת "פעולות" הישנה
  לתוך תא "ספק".
- בורר קישור לתנועה ובורר מיון D8 — עברו מתוך תא בטבלה לפופאפים נפרדים,
  נפתחים מסרגל הפעולות.
