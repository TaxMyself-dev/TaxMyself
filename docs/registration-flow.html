תשלום — עתידי (ספק תשלומים)
GET /auth/billing-statusמחזיר finalAmountNis לחיוב

חישוב הסכום

INVOICES בלבד → 15 ₪
OPEN_BANKING בלבד → 45 ₪
שניהם → 54 ₪ (הנחת combo)
+ discountPercent אישי → finalAmountNis

לאחר תשלום מוצלח

payStatus → PAID
modulesAccess → += המודול
SubscriptionGuard → ✅ מאשר גישה
★
סיכום — אינדיקציות קריטיות
שדה / ישות משמעות סוג
businessStatus קובע מה מוצג בממשק (כרטיסי ניווט) UI
hasOpenBanking האם המשתמש חיבר בנקאות פתוחה (טבלת תנועות + באנר) UI
modulesAccess[] Gate ל-API — מה מותר לגשת, מתעדכן יומית הרשאות
UserModuleSubscription מתי ה-trial של כל מודול מסתיים תשלום
payStatus (per-module) TRIAL / PAYMENT_REQUIRED / PAID תשלום
discountPercent + discountLabel הנחה אישית — מוגדר ידנית ב-DB תשלום
finalAmountNis הסכום הסופי לחיוב אצל ספק תשלומים תשלום
