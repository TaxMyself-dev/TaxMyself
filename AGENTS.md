## Backend (`backend/src/`)

- [accountant-tasks](backend/src/accountant-tasks/AGENTS.md) — accountant task/todo list, auto-generates recurring VAT/advance-tax/annual-report tasks
- [annual-report](backend/src/annual-report/AGENTS.md) — annual tax report workflow: questionnaire, document uploads, finish/reported lifecycle
- [auth](backend/src/auth/AGENTS.md) — not an active module; real auth logic lives in `guards/firebase-auth.guard.ts` and `users/auth.service.ts`
- [billing](backend/src/billing/AGENTS.md) — subscription billing: plans, trials, CardCom checkout/webhooks, renewals, receipts
- [bookkeeping](backend/src/bookkeeping/AGENTS.md) — double-entry bookkeeping core: chart of accounts, journal entries/lines
- [business](backend/src/business/AGENTS.md) — Business entity: identity, VAT/tax config, SHAAM tokens, Drive folder linkage
- [cities](backend/src/cities/AGENTS.md) — static Israeli cities reference data
- [clients](backend/src/clients/AGENTS.md) — a business owner's own customer/vendor contact list (not the accountant↔client relationship — see `delegation`)
- [cloud](backend/src/cloud/AGENTS.md) — search endpoint over expenses with attached files
- [delegation](backend/src/delegation/AGENTS.md) — accountant-client permission relationships, invitations, accountant-created client accounts
- [demo-data](backend/src/demo-data/AGENTS.md) — seeds/resets full demo user accounts across most modules
- [documents](backend/src/documents/AGENTS.md) — issued-document lifecycle (invoices/receipts/credit notes) + inbound OCR pipeline
- [expenses](backend/src/expenses/AGENTS.md) — expenses, category/sub-category taxonomy, suppliers, journal posting
- [feezback](backend/src/feezback/AGENTS.md) — Feezback Open Banking (AISP) integration
- [finsite](backend/src/finsite/AGENTS.md) — Finsite bookkeeping-data API integration (possibly legacy — see flags)
- [google-drive](backend/src/google-drive/AGENTS.md) — Google Drive API wrapper for per-user/business folder hierarchy
- [mail](backend/src/mail/AGENTS.md) — transactional email via Brevo
- [notifications](backend/src/notifications/AGENTS.md) — log-only notification side-channel for report-workflow status changes
- [report-workflow](backend/src/report-workflow/AGENTS.md) — client-facing VAT/advance-tax report obligation lifecycle
- [reports](backend/src/reports/AGENTS.md) — generates VAT/P&L/ledger/advance-tax/depreciation/uniform-file reports
- [shaam](backend/src/shaam/AGENTS.md) — Israel Tax Authority SHAAM OpenAPI integration (OAuth2 + invoice approval)
- [transactions](backend/src/transactions/AGENTS.md) — bank/card transaction ingestion, classification, slim/cache dual-table model
- [users](backend/src/users/AGENTS.md) — user entity, signup/signin, Firebase auth, Drive folder provisioning
- [shared](backend/src/shared/AGENTS.md) — date/VAT-period math, generic repository lookup, FX-rate (BOI) caching
